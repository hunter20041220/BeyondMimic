Level C should not start as long training until these inputs are available or generated:

- Motion-tracking teacher policies or exported teacher actions.
- Rollout dataset of state-action pairs generated by motion-tracking policies.
- VAE DAgger dataset and collection protocol.
- State-latent trajectory dataset with root pose/twist, local body positions/velocities, and latent actions.
- Guidance cost definitions for joystick velocity, waypoint, keyframe inpainting, and SDF obstacle avoidance.
- Evaluation metrics matching Fig. 5, Fig. 6, and latent diffusion ablation claims.

## Recommended Local Reimplementation Stages

1. Complete Level B tracking smoke, export at least one valid motion-tracking ONNX, and validate its metadata.
2. Define a minimal PyTorch `bm_diffusion` environment only after the tracking rollout format is fixed.
3. Implement dataset schemas first, with small synthetic fixtures and shape tests.
4. Implement conditional VAE and train a tiny smoke model on synthetic or one-motion rollout data.
5. Implement state-latent transformer diffusion with the paper hyperparameters.
6. Implement guidance costs as separate, unit-tested functions.
7. Run tiny smoke training/inference before any long VAE/diffusion run.
