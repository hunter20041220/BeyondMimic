## Effect on English Reading Report

This strengthens the reproduction limitations section: the simulator problem is now localized to the USD write/runtime layer rather than an asset-path mistake. It supports a concrete reproducibility discussion about hidden simulator state, USD layer save policy, GPU/Vulkan interaction, and why reproducing robotics papers can fail before any learning algorithm is executed.

## Next Step

Investigate Kit/USD save policy directly. Candidate next probes are: try `.usd` vs `.usda` save under multiple project-local cache paths, inspect Sdf layer permission flags, try the Isaac Sim Python runtime outside AppLauncher if available, and test whether a non-renderer or CPU/minimal Kit app can save a USD layer. A successful minimal stage save should precede any further official replay attempt.
